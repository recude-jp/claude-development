# CLAUDE.md — このリポジトリでのClaude向け操作指示

## 役割

このリポジトリは「初心者がClaudeと対話しながら要件定義・基本設計を進め、有識者がGitHub経由でレビューする」プロセスを管理するものです。

初心者はGitの操作を知らないため、**GitHubへの保存・共有はClaudeが代行**します。

---

## git commit/push の手順

初心者から「共有してください」「GitHubに送ってください」等の依頼を受けたら、以下の手順を実行してください。

### 1. 変更確認

```bash
git status
git diff --stat
```

### 2. ステージング・コミット・プッシュ

```bash
git add -A
git commit -m "<コミットメッセージ>"
git push origin main
```

### 3. 完了報告

「GitHubに共有しました。有識者に声をかけてください」と初心者に伝えてください。

---

## コミットメッセージ規約

フォーマット: `<種別>(<ステップ>): <内容>`

| 種別 | 使いどころ |
|---|---|
| `feat` | 各ステップのファイルを新規作成・初回記入完了したとき |
| `update` | 既存ファイルを追記・修正したとき(レビュー反映など) |
| `fix` | 誤記・軽微な修正 |
| `docs` | READMEやCHANGELOGなどの管理ファイル更新 |

### 例

```
feat(①): 要件ブレスト完成
update(②): 有識者コメントを反映し基本設計を修正
feat(②): 基本設計 初版完成
update(③): レビューチェックリスト記入
feat(④): 実装指示書 完成
docs: README_for_beginner.md 更新
```

## ブランチ運用

- 基本は `main` ブランチで直接作業します
- 有識者がレビュー用ブランチを切る場合は別途指示があります

---

## よくある依頼フレーズと対応

| 初心者の依頼 | Claudeの操作 | コミットメッセージ例 |
|---|---|---|
| 「①の要件ブレストができました。有識者に共有してください」 | `01_requirements_brainstorm.md` を add/commit/push | `feat(①): 要件ブレスト完成` |
| 「②の基本設計ができました。有識者に共有してください」 | `02_basic_design.md` を add/commit/push | `feat(②): 基本設計 初版完成` |
| 「②を修正しました。再共有してください」 | 変更ファイルを add/commit/push | `update(②): 有識者コメントを反映` |
| 「保存してください」「共有してください」 | 変更ファイル全体を add/commit/push | 文脈に応じたメッセージを使用 |

---

## エラー時の対応

- `git push` が失敗した場合は、エラーメッセージをそのまま初心者に見せ、「有識者に画面ごと共有してください」と伝えてください
- SSH鍵の問題など環境依存のエラーは自己解決しようとせず、有識者に委ねてください
