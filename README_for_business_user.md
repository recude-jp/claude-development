# このプロジェクトの進め方

| | |
|---|---|
| 業務ユーザー | <!-- BUSINESS_USER_NAME --> |
| システムアドバイザー | <!-- ADVISOR_INFO --> |

**作業はすべて Claude Cowork の画面の中で完結します。**
ファイルを直接触ったり、難しいアプリを操作する必要はありません。

---

## 作業の全体像

```
① 今の業務を話す → ② 設計を考える → ③ アドバイザーのチェック → ④ コードを作る
```

各ステップは Claude との会話で進みます。
ステップが終わるたびに Claude が「共有しますか？」と確認してくれます。

---

## 始め方（毎回の作業）

1. **Claude Cowork を開く**
2. **システムアドバイザーに教えてもらったフォルダを選択する**
3. **以下のように話しかける**

> 「業務を自動化したいです」

あとは Claude の質問に答えていくだけです。
「何を用意すればいいか」「どのファイルを開くか」などは Claude が案内します。

---

## 各ステップで Claude がやってくれること

| ステップ | Claude がやること | あなたがやること |
|---|---|---|
| ① 要件整理 | 業務の流れを質問しながら整理し、ファイルに書き込む | 今の業務を話す |
| ② 基本設計 | システムの構成案を提示し、対話で設計書を作る | 希望や不安を伝える |
| ③ アドバイザーチェック | （Claudeは待機） | アドバイザーからの連絡を待つ |
| ④ 実装 | コードを作成し、使い方をまとめた README を生成する | 試してみて感想を伝える |

---

## システムアドバイザーへの共有方法

各ステップが終わったら、Claude にこう伝えるだけです：

> 「〇〇ができました。システムアドバイザーに共有してください」

Claude が自動で保存・送信まで行います。
GitHubの操作は一切不要です。

---

## 困ったときは

| 状況 | 対処 |
|---|---|
| Claude の言っている意味がわからない | 「もっとかんたんに説明してください」と伝える |
| 話が行き詰まった・うまく進まない | システムアドバイザーに Cowork の画面を見せて相談する |
| エラーメッセージが出た | 画面をそのままシステムアドバイザーに共有する（自分で解決しなくてOK） |

「わからないことを聞く」のはこのプロセスの正式な一部です。
遠慮せず、早めに確認してください。
